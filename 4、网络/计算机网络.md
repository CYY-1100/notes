# http 是什么？
http是一种无状态, 无连接, 非持久化 的 单向半双工 应用层协议
## 什么是无状态？
- 对于历史连接是完全没有记忆的, 每一次连接都是新的连接

## 什么是无连接？
-  一次请求, 一次响应, 不会持续.

## 什么是非持久化？
-  一次请求, 一次响应, 不会持续.

## 什么是单向半双工？
- 请求只能由客户端发起, 服务端只能对于请求做出应答, 服务端不能主动地向客户端发送数据

# http有什么特性
- 简单快速
- 灵活 
- 无连接
- 无状态
- 基于请求/响应模型
- 基于 TCP/IP 
- 明文传输 (不安全)

# https 是什么？
- http 加入了 SSL/TLS 协议

# HTTP 与 HTTPS 的区别？
- 安全性：HTTP 是明文传输，不安全；HTTPS = HTTP + SSL/TLS，数据加密传输。
- 端口：HTTP 默认 80，HTTPS 默认 443。
- 证书：HTTPS 需要向 CA 机构申请数字证书（涉及公钥/私钥机制）。
- 性能：HTTPS 由于握手过程，初次连接稍慢，但支持 HTTP/2 后性能往往优于 HTTP/1.1。
- SEO：搜索引擎通常优先收录 HTTPS 网站。

# 简述 HTTPS 的握手过程？
客户端发送 ClientHello（支持的加密套件、随机数）。
服务端返回 ServerHello（选定的加密套件、随机数）、证书、ServerKeyExchange 等。
客户端验证证书合法性，生成预主密钥（用服务器公钥加密），发送给服务器。
双方利用随机数和预主密钥生成会话密钥。
后续通信使用会话密钥进行对称加密。

2xx (成功): 200 OK (正常), 204 No Content (无返回值), 206 Partial Content (断点续传)。
3xx (重定向):

# 版本演进与性能优化
## HTTP/1.1 与 HTTP/2 的主要区别？（性能优化核心）
多路复用 (Multiplexing)：HTTP/2 在同一 TCP 连接上并行发送多个请求/响应，解决了 HTTP/1.1 的队头阻塞问题。
二进制分帧：HTTP/2 采用二进制格式解析，更高效，不再依赖纯文本。
头部压缩 (HPACK)：大幅减少请求头体积，降低带宽消耗。
服务器推送 (Server Push)：服务器可主动推送资源给客户端（但在实际应用中因复杂性常被禁用或谨慎使用）。

## 了解 HTTP/3 (QUIC) 吗？它解决了什么问题？
基于 UDP：不再依赖 TCP，解决了 TCP 层面的队头阻塞（丢包重传时阻塞整个连接）。
快速握手：0-RTT 或 1-RTT 建立连接，提升首屏速度。
连接迁移：网络切换（如 WiFi 切 4G）时不需要重新握手，保持连接不断。
