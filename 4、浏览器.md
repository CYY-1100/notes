# 浏览器从输入域名到页面展示都干了些什么
1. 首先会判断你输入的是不是 URL
2. DNS 查询
    1. 浏览器缓存：检查最近是否访问过该域名,有则直接返回缓存的 IP。
    2. 操作系统缓存：检查本地 hosts 文件是否有对该域名的映射。
    3. 路由器缓存：询问你的路由器是否有缓存。
    4. ISP DNS 服务器：网络服务提供商（ISP）的 DNS 服务器。
3. 建立 TCP 连接（及 TLS 握手）
4. 发送 HTTP 请求
5. 服务器处理并返回响应
6. 浏览器解析和渲染页面
    1. 接收 HTML 并构建 DOM 树
    2. 解析 CSS 并构建 CSSOM 树
    3. 构建渲染树。将前两者合并起来，创建一个包含所有可见元素及其计算样式的树
    4. 布局 将布局好的渲染树转换成屏幕上的实际像素
    5. 合成 多个图层, 按照正确的层级合成在一起，显示在屏幕上。
7. 加载额外资源 
    - 例如图片，览器会再次向服务器发起请求，获取这些资源
    - JS 可能会修改 DOM 和 CSSOM，从而触发重新进行“构建渲染树 -> 布局 -> 绘制”的流程。
