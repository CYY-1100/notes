# 浏览器

## 初级

### 从输入 URL 到页面展示
1. URL 判断 → 2. DNS 查询 → 3. TCP/TLS 连接 → 4. HTTP 请求 → 5. 服务器响应 → 6. 解析渲染 → 7. 加载资源

### DNS 查询
域名解析为 IP 地址。
- 查询顺序：浏览器缓存 → hosts 文件 → 路由器 → ISP DNS

### HTTP 状态码
| 状态码 | 含义 |
|--------|------|
| 200 | 成功 |
| 301/302 | 重定向 |
| 304 | 协商缓存 |
| 400 | 请求错误 |
| 404 | 资源不存在 |
| 500/503 | 服务器错误 |

### 渲染流程
解析 HTML → DOM 树 → 解析 CSS → CSSOM 树 → 渲染树 → 布局 → 绘制 → 合成

### 浏览器存储
| 类型 | 容量 | 说明 |
|------|------|------|
| Cookie | 4KB | 随请求发送 |
| LocalStorage | 5MB+ | 跨标签页 |
| SessionStorage | 5MB+ | 当前标签页 |
| IndexedDB | 很大 | 结构化数据 |

### 浏览器内核
| 内核 | 浏览器 |
|------|--------|
| Blink | Chrome, Edge, Opera |
| Webkit | Safari |
| Gecko | Firefox |
| Trident | IE（已淘汰） |

## 中级

### DNS 详解
- **TTL**：缓存存活时间
- **记录类型**：A（IPv4）、AAAA（IPv6）、CNAME、MX

### TCP 三次握手
```
客户端 ──── SYN ────→ 服务端
客户端 ←── SYN-ACK ←── 服务端
客户端 ──── ACK ────→ 服务端
```

### TLS 握手（HTTPS）
Client Hello → Server Hello（证书）→ 证书验证 → 密钥协商 → 加密通道

### 回流与重绘
| 类型 | 触发条件 | 影响 |
|------|----------|------|
| 回流 | 大小、位置变化 | **严重** |
| 重绘 | 颜色、背景变化 | 较轻 |

**优化**：批量 DOM 操作、transform/opacity、读写分离、will-change

### 缓存策略
| 类型 | 响应头 | 特点 |
|------|--------|------|
| 强缓存 | Cache-Control, Expires | 不发请求 |
| 协商缓存 | ETag, Last-Modified | 发请求验证 |

### 浏览器 API
| API | 用途 |
|-----|------|
| requestAnimationFrame | 动画帧 |
| IntersectionObserver | 懒加载、滚动监听 |
| MutationObserver | DOM 变化监听 |
| ResizeObserver | 元素尺寸监听 |
| Storage 事件 | 跨标签页通信 |

### HTTP 版本
| 版本 | 请求方式 | 特点 | 缺点 |
|------|----------|------|------|
| HTTP/1.0 | 短连接，每次新建 | 简单 | 连接频繁 |
| HTTP/1.1 | 长连接复用 | keep-alive、管道化 | 队头阻塞 |
| HTTP/2 | 单连接多路复用 | 多路复用、头部压缩、服务器推送 | TCP 队头阻塞 |
| HTTP/3 | QUIC 协议 | 无队头阻塞、0-RTT | 部署复杂 |

## 高级

### 垃圾回收（GC）

#### 算法
| 算法 | 说明 |
|------|------|
| 标记-清除 | 从根对象标记，清除未标记对象 |
| 引用计数 | 计数为 0 时回收 |
| 标记-整理 | 整理内存，解决碎片 |

#### V8 GC 策略
- **分代回收**：新生代 + 老生代
- **Scavenge**：新生代，快速
- **Mark-Compact**：老生代，整理内存
- **增量标记**：分批执行，避免卡顿

### 内存泄漏

#### 常见原因
| 原因 | 说明 |
|------|------|
| 全局变量 | 未声明的变量挂在 window |
| 闭包 | 持有外部变量引用 |
| 定时器 | setInterval 未清理 |
| 事件监听 | addEventListener 未移除 |
| DOM 引用 | 删除后变量仍引用 |

#### 排查与优化
- DevTools → Memory 快照对比
- Performance Monitor
- 及时清理定时器/监听
- 使用 WeakMap/WeakSet

### 关键渲染路径（CRP）

#### 阻塞资源
| 资源 | 默认行为 | 优化 |
|------|----------|------|
| CSS | 阻塞渲染 | 内联关键 CSS |
| JS | 阻塞 DOM | defer/async |

#### 优化方法
- 减少关键资源数量
- 减少关键字节大小
- 关键 CSS 内联
- 资源预加载/预取

### 事件循环

| 任务类型 | 示例 |
|----------|------|
| 宏任务 | setTimeout, setInterval, I/O, UI 渲染 |
| 微任务 | Promise.then, MutationObserver, queueMicrotask |

**执行顺序**：主线程 → 微任务 → 宏任务

```js
async function test() {
  console.log(1);
  await Promise.resolve();
  console.log(2); // 微任务
}
test();
console.log(3);
// 输出：1, 3, 2
```

### 浏览器安全

#### 同源策略
协议、域名、端口相同才算同源。

#### CORS
| 响应头 | 说明 |
|--------|------|
| Access-Control-Allow-Origin | 允许的源（* 或具体域名） |
| Access-Control-Allow-Methods | 允许的请求方法（GET, POST 等） |
| Access-Control-Allow-Headers | 允许的请求头 |
| Access-Control-Allow-Credentials | 是否允许携带 cookie |
| Access-Control-Max-Age | 预检请求缓存时间 |

#### XSS
- 类型：反射型、存储型、DOM 型
- 防范：输入过滤、输出转义、CSP

#### CSRF
- 防范：CSRF Token、SameSite Cookie

### Web Worker

#### 是什么
- 后台 JavaScript 线程
- 与主线程隔离，不阻塞 UI
- 无法访问 DOM、window

#### 类型
| 类型 | 说明 |
|------|------|
| Dedicated Worker | 专用，单页面使用 |
| Shared Worker | 共享，多页面共用 |
| Service Worker | PWA 缓存/离线（生命周期：install → activate → fetch → message） |

#### 通信
```js
// main.js
const worker = new Worker('worker.js');
worker.postMessage({ data: 100 });
worker.onmessage = e => console.log(e.data);

// worker.js
self.onmessage = e => self.postMessage(e.data * 2);
```

#### 场景
复杂计算、数据处理、加密解密、图像处理、文件压缩
