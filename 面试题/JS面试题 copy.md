### 9. 说一说类的继承
基于其原型链
底层原理：寄生组合式继承
| 继承方式 | 核心思路 | 优点 | 缺点 |
| :--- | :--- | :--- | :--- |
| 原型链继承 | `Child.prototype = new Parent()` | 简单，可复用父类方法 | 引用类型属性会被所有实例共享；无法向父类传参 |
| 构造函数继承 | `Parent.call(this, ...)` | 属性独立，可传参 | 无法继承父类原型上的方法，方法无法复用 |
| 组合继承 | 原型链 + 构造函数 | 方法复用 + 属性独立 | 父类构造函数被调用两次，造成性能浪费 |
| 寄生组合继承 | `call` + `Object.create` | 完美方案：高效、灵活、无冗余 | 代码相对繁琐（ES6 已封装） |
